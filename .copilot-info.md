# Copilot用プロジェクト情報

## プロジェクト概要

- **目的**: 自分のプロフィールページを作成・公開
- **フレームワーク**: Next.js
- **公開先**: GitHub Pages
- **デプロイ方法**: GitHub Actions による自動デプロイ
- **ビルド方式**: 静的サイト生成（Static Site Generation）

## 技術スタック

- Next.js（静的出力設定必要）
- TypeScript
- GitHub Actions（CI/CD）
- GitHub Pages

## 設計方針

- **データ管理**: constantファイルでプロフィール情報を一元管理
- **更新性重視**: 設定変更時のコード修正を最小限に
- **型安全性**: TypeScriptで型定義を厳格に管理

## 作業ステップ

1. **データ構造設計**: constantファイルとTypeScript型定義の作成
2. **Next.js設定**: 静的出力用に設定
3. **プロフィール実装**: 定数ファイルを活用したコンポーネント作成
4. **GitHub Actions**: ワークフロー設定
5. **GitHub Pages**: デプロイ設定

## ファイル構成予定

```
constants/
  ├── profile.ts      // 基本プロフィール情報
  ├── skills.ts       // スキル・技術スタック
  ├── experience.ts   // 職歴・経験
  └── index.ts        // 全体のexport
types/
  └── profile.ts      // 型定義
```

## 重要な設定

- `next.config.ts`: `output: 'export'` を設定
- GitHub Actions: `pages-build-deployment` ワークフロー
- 出力ディレクトリ: `out/`

このファイルはAIアシスタント（GitHub Copilot）がプロジェクトの状況を把握しやすくするためのメモです。
